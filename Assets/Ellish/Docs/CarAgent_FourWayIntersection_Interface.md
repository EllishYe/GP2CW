# Car Agent Four-Way Intersection Interface

This interface is generated by `CrosswalkGenerator` after roads and walkable/crosswalk data are generated.

## Where to Read

```csharp
CrosswalkGenerator crosswalkGenerator;
List<FourWayIntersectionData> intersections = crosswalkGenerator.GetFourWayIntersections();
```

Lane path data still comes from:

```csharp
RoadNetworkGenerator roadNetworkGenerator;
List<AgentLaneData> lanes = roadNetworkGenerator.GetAgentLanes();
```

## Lane Fields

Use these fields for car movement:

```csharp
AgentLaneData.startPoint
AgentLaneData.endPoint
AgentLaneData.vehicleStartPoint
AgentLaneData.vehicleStopPoint
```

- `startPoint` / `endPoint`: original lane geometry points near the junction.
- `vehicleStartPoint` / `vehicleStopPoint`: adjusted points outside the crosswalk area. Cars should stop at `vehicleStopPoint` before entering an intersection.

## Four-Way Intersection Fields

Each `FourWayIntersectionData` represents one four-way junction.

```csharp
int id;
int junctionId;
Vector3 center;
List<FourWayIntersectionArmData> arms;
```

Each intersection has 4 `arms`.

```csharp
int armIndex;
int axisGroupIndex;
int crosswalkId;
int connectedEdgeId;
RoadKind roadKind;
Vector3 outwardDirection;
Vector3 innerCenter;
Vector3 outerCenter;
List<int> incomingLaneIds;
List<int> outgoingLaneIds;
```

## Suggested Traffic-Light Grouping

Use `axisGroupIndex` for the first traffic-light version:

- `axisGroupIndex == 0`: one pair of opposite arms.
- `axisGroupIndex == 1`: the other pair of opposite arms.

When group 0 has green light, allow cars whose lane id is in group 0 arm `incomingLaneIds`.
When group 1 has green light, allow cars whose lane id is in group 1 arm `incomingLaneIds`.

`outgoingLaneIds` can be used later for turning logic or lane connection rules.

## Notes

- Only four-way intersections appear in `GetFourWayIntersections()`.
- Two-way and three-way crosswalks remain visual-only and are not included in this interaction interface.
- The current `junctionId` and `connectedEdgeId` are runtime ids, suitable for one generated road network session.
